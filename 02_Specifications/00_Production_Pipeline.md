# HISTORIUM — PRODUCTION PIPELINE

**Versión:** 1.0

**Estado:** Oficial

---

# Filosofía

Historium no produce contenido.

Construye conocimiento.

Todo elemento del universo sigue exactamente el mismo proceso de producción.

No existen excepciones.

---

# ETAPA 1

## Investigación

Antes de escribir una sola palabra se responde:

¿Qué pregunta histórica resolverá esta campaña?

¿Qué periodo abarca?

¿Qué personajes participan?

¿Qué conceptos introduce?

¿Qué campañas conecta?

# ETAPA 5

## Capítulos

Ahora se diseña cada capítulo.

No los nodos.

Solo los capítulos.

Cada capítulo responde:

¿Qué cambia?

¿Qué aprende el jugador?

¿Qué entidades evolucionan?

¿Cómo termina?

# ETAPA 6

## Nodos

Solo ahora comienza la escritura.

Cada nodo utiliza:

Entidades existentes.

Relaciones existentes.

Museo.

Sistema RPG.

Recompensas.

No inventa estructura.

Solo cuenta una historia.

# Regla de Oro

Nunca escribir un nodo antes de diseñar la campaña.

Nunca diseñar una campaña antes de crear las entidades.

Nunca crear entidades sin conocer su función dentro del universo.

Historium no improvisa.

Historium construye.

1. Idea

Toda campaña inicia con una pregunta histórica.

Ejemplo:

¿Cómo una ciudad llamada Roma conquistó el Mediterráneo?

Una campaña nunca inicia con:

Hagamos una campaña de Roma.

Primero debe existir una pregunta poderosa.

2. Investigación

Antes de escribir nodos se define:

periodo histórico,
fuentes principales,
personajes centrales,
lugares clave,
eventos principales,
conceptos que el jugador debe comprender,
debates históricos relevantes,
campañas conectadas.

Producto generado:

Research.md

Antes de escribir nodos se crea la arquitectura completa de la campaña.

arco narrativo,

final de campaña,
campañas conectadas.

6. Chapter Spec

Cada capítulo debe representar una transformación histórica.

No se divide por cantidad de nodos.

Se divide por cambio histórico.

Ejemplo:

Capítulo 2 — Atenas despierta
Transformación: Atenas pasa de crisis aristocrática a laboratorio político.
7. Node Spec

Cada nodo es una unidad jugable.

Debe tener:

hook,
desarrollo narrativo,
conflicto,
museo actualizado,
entidades relacionadas,
decisión histórica,
quiz,
curiosidades,
cierre con cliffhanger.

Duración ideal:

15–20 minutos
8. Curiosities

Las curiosidades no repiten el nodo.

Amplían el mundo.

Deben incluir:

contexto adicional,
debates,
detalles sorprendentes,
conexiones modernas,
contenido premium.
9. Quiz

El quiz no mide memoria.

Mide comprensión.

Debe preguntar:

causas,
consecuencias,
decisiones,
relaciones,
impacto histórico.
10. Prompts IA

Cada nodo debe generar prompts para:

imagen hero,
retrato,
mapa,
carta/reliquia,
audio,
video corto,
miniatura.
11. Validación

Todo contenido debe pasar revisión en seis niveles:

Histórica

¿Es correcto?

Narrativa

¿Es emocionante?

Pedagógica

¿Se entiende?

Museo

¿Qué entidades actualiza?

Técnica

¿El Markdown puede importarse?

Visual

¿Tiene potencial gráfico?

12. Importación

El contenido final debe estar en Markdown estructurado con frontmatter YAML.

Ejemplo:

id: GRC-008
type: node
campaign: CAMP_GREECE_CLASSICAL
chapter: GRC-CH02
entities_updated:
  - CHAR_000001
  - CITY_000001
  - CONCEPT_000001
13. Publicación

Después de importarse, el contenido debe aparecer en:

campaña,
museo,
mapa,
colecciones,
buscador,
grafo de conocimiento,
perfil del jugador.
Regla de oro

Nunca escribir un nodo antes de conocer:

qué campaña alimenta,
qué capítulo transforma,
qué entidades usa,
qué relaciones crea,
qué parte del Museo hace crecer.

Si no puede responder eso, el nodo todavía no debe escribirse.

# Arquitectura Canónica

Esta especificación forma parte del Canon Oficial de Historium.

Debe mantener compatibilidad con el Museo, el Grafo Histórico, el pipeline de importación y las campañas narrativas.

# Assets

Todo asset debe explicar, ubicar o intensificar el contenido.

No se aceptan assets decorativos sin función narrativa o museística.

---

# Versionado

Todo cambio relevante debe actualizar versión y nota editorial.

El Canon debe poder rastrear por qué cambió una entidad, nodo, campaña o documento.

---

# Objetivo

Establecer una regla de producción clara para que este contenido pueda escribirse, revisarse, importarse y expandirse sin perder coherencia con el Canon Oficial.

---

# Principios

- Debe alimentar el Museo Vivo.
- Debe conectarse con el Grafo Histórico.
- Debe poder vivir fuera de una sola campaña.
- Debe sostener expansión futura.
- Debe mantener tono premium, claro y cinematográfico.

---

# Museo

Toda pieza producida con esta especificación debe declarar qué parte del Museo hace crecer.

Debe indicar si desbloquea descubrimiento, estudio, dominio o maestría.

---

# Checklist

- [ ] Tiene identidad única.
- [ ] Hace crecer el Museo.
- [ ] Incluye relaciones útiles.
- [ ] Respeta voz y tono Historium.
- [ ] Puede importarse sin ambigüedad.
- [ ] Está listo para revisión editorial.

---

# Regla Suprema

Si esta pieza no aumenta comprensión histórica dentro del Museo Vivo, debe rediseñarse antes de entrar al Canon.
