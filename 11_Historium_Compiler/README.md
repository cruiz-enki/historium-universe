# HISTORIUM — COMPILER

**Versión:** 1.0  
**Estado:** Oficial  
**Tipo:** Compiler Architecture

---

# Propósito

Historium Compiler es el motor conceptual que convierte el Canon Oficial en contenido jugable.

Toma Markdown, YAML, entidades, relaciones, reglas editoriales y reglas de IA.

Luego produce campañas, nodos, prompts, validaciones, commits e importaciones.

No reemplaza al editor.

No reemplaza al Canon.

Organiza el proceso para que la IA no improvise.

# Conexiones

## 00_Biblia_Editorial

Define voz, tono, narrativa y guía visual.

El Compiler la usa para limitar el estilo de salida.

## 01_Game_Design

Define progresión, recompensas y sistema de conocimiento.

El Compiler lo usa para que cada output alimente el Museo.

## 02_Specifications

Define contratos de campaña, capítulo, nodo y entidades.

El Compiler los usa como schemas editoriales.

## 03_Knowledge

Contiene entidades, Registry y Relationship Types.

El Compiler lo usa para resolver IDs y relaciones.

## 09_Architecture

Define naming, frontmatter, lifecycle, versioning, importer, Supabase y Museo.

El Compiler lo usa como arquitectura base.

## 10_AI

Define reglas de producción, validación, roles y prohibiciones para IA.

El Compiler lo usa para construir prompts seguros.

## 06_Importer

Recibe Markdown validado y lo convierte en datos activos.

El Compiler prepara output compatible.

## Supabase

Recibe entidades, relaciones, campañas, nodos, assets y progreso operativo.

## App Historium

Presenta la experiencia final al jugador.

---

# Regla Suprema

El Compiler no produce contenido.

El Compiler produce contexto correcto para que el contenido pueda existir sin romper el universo.
