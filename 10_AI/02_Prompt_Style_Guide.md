# HISTORIUM — PROMPT STYLE GUIDE

**Versión:** 1.0  
**Estado:** Oficial

---

# Propósito

Este documento define cómo escribir prompts para generar contenido Historium con IA.

Un buen prompt no pide “haz una lección”.

Pide construir una pieza del Museo Vivo.

---

# Estructura de prompt

Todo prompt debe incluir:

- contexto,
- objetivo,
- tono,
- formato,
- entidades obligatorias,
- restricciones,
- output esperado,
- checklist.

---

# Contexto

Explica dónde vive la pieza dentro del universo.

Ejemplo:

```text
Esta pieza pertenece a Greece_Classical y debe desbloquear el primer entendimiento de la polis.
```

---

# Objetivo

Define qué debe entender el jugador.

No basta con “explicar Atenas”.

Debe decir qué cambia en su Museo.

---

# Tono

Usar:

- premium,
- cinematográfico,
- claro,
- museístico,
- emocionalmente preciso.

Evitar:

- escolar,
- genérico,
- enciclopédico,
- infantil.

---

# Formato

Indicar Markdown, frontmatter, secciones y YAML esperado.

---

# Entidades obligatorias

Listar IDs y nombres.

La IA debe usarlos.

No debe crear sustitutos.

---

# Restricciones

Indicar:

- no inventar fuentes,
- no duplicar entidades,
- no usar relaciones libres,
- no mezclar épocas,
- no afirmar hipótesis como certeza.

---

# Output esperado

Describir exactamente qué debe devolver.

---

# Checklist

Todo prompt debe pedir autovalidación final.
